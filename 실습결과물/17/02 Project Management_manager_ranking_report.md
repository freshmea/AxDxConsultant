# Project Management 매니저 성과 1페이지 요약 보고서

## 목적
이 보고서는 `02 Project Management_ml_imputed.xlsx`를 기준으로 매니저별 완료 성과를 비교하고, 2025년 8월 말 기준으로 아직 완료 시점이 오지 않은 작업 때문에 평가가 왜곡되지 않도록 보정한 뒤 석차를 제시한다.

## 보정 알고리즘
- 기준 시점: `2025-08-31`
- 평가 대상 분모: `Planned End Date <= 2025-08-31` 인 작업만 포함
- 보정 완료율: `Completed / eligible_tasks`
- 보정 미완료율: `1 - 보정 완료율`
- 보조 지표: `active_overdue_rate = (Delayed + In Progress + Not Started + On Hold) / eligible_tasks`
- 정렬 규칙: `보정 완료율` 내림차순, 동률이면 `active_overdue_rate` 오름차순, 그다음 `평균 진행률` 내림차순

## 보정이 필요한 이유
- 단순 완료율은 아직 종료 예정일이 오지 않은 작업까지 실패처럼 보이게 만들 수 있다.
- 그래서 원칙적으로는 8월 말까지 끝났어야 하는 작업만 평가해야 공정하다.
- 다만 이번 데이터는 `Planned End Date` 최대값이 `2025-08-23`이라 8월 이후 종료 예정 작업이 없다.
- 즉 이번 보정은 방법론상 필요하지만, 실제 결과는 전체 작업 기준 석차와 동일하다.

## 상위권과 하위권 특징 비교
- 상위 5명 평균: 완료율 `55.1%`, 활성 미완료율 `35.2%`, 평균 진행률 `67.4%`
- 하위 5명 평균: 완료율 `37.0%`, 활성 미완료율 `47.9%`, 평균 진행률 `56.1%`
- 상위권은 하위권보다 평균 완료 수가 많고, 지연·보류·미착수로 남아 있는 작업 비율이 낮다.
- 상위권은 예산 규모가 특별히 작은 것도 아니다. 상위 5명 평균 예산은 `1.35M`, 하위 5명은 `1.47M`으로, 단순히 쉬운 포트폴리오만 맡았다고 보기는 어렵다.
- 하위권은 `Delayed`, `Not Started`, `In Progress`가 누적되는 패턴이 강하다. 즉 착수-진행-종결 전환 속도가 느리다.
- 특히 `Marcus Johnson`, `Jennifer Lee`, `Robert Garcia`는 완료율이 낮은데도 담당 예산이 큰 편이라 운영 리스크가 상대적으로 크다.

## 해석
- 잘 관리하는 매니저는 완료율이 높을 뿐 아니라 미완료 작업을 오래 끌지 않는다.
- 관리가 약한 매니저는 작업을 많이 맡아서가 아니라, 진행 중과 지연 상태를 해소하지 못해 backlog가 쌓이는 경향이 더 크다.
- 따라서 평가 포인트는 “얼마나 많이 맡았는가”보다 “끝나야 할 작업을 얼마나 제때 닫았는가”에 두는 것이 맞다.

## 운영 플랜
- 상위 5명 운영 방식 표준화: 주간 리스크 점검 방식, 일정 재조정 시점, 종결 체크리스트를 문서화해서 중하위권 매니저에게 공통 적용한다.
- 하위 5명 집중 코칭: `Delayed + In Progress + Not Started` 비중이 높은 이유를 개별 리뷰하고, 2주 단위로 backlog 감축 목표를 부여한다.
- 고예산 하위권 모니터링: `Robert Garcia`, `Jennifer Lee`, `Marcus Johnson`은 예산 영향이 커서 별도 임원 리뷰 대상로 두는 것이 합리적이다.
- 성과 관리 기준 이원화: 완료율만 보지 말고 `활성 미완료율`을 함께 KPI로 둬서 “끝내는 능력”과 “밀리는 작업 억제”를 동시에 본다.
- 월간 재배분 규칙 도입: 완료율 상위권에는 전략성 높은 신규 과제를 배정하고, 하위권에는 기존 backlog 해소 전까지 신규 투입을 제한한다.
