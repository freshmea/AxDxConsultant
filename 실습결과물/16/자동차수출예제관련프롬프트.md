# 자동차 수출 관련 예제 프롬프트

---

```text
이반칸을 채우려고 하는데 자동차 수출에 관련되 csv 파일을 가지고 적절한 내용을 채워줘

plotly 라이브러리를 최대한 활용해서 보고서를 작성 하고자 하는데, https://velog.io/@sh070928/Plotly-Basic-Chart-%EB%B0%B0%EC%9A%B0%EA%B8%B0 이 글을 참고해서 넣을 수 있는 그래프를 최대한 활용하여 분석 결과를 다양하게 작성할 수 있도록 프롬프트를 만들어줘

너는 자동차 산업 데이터 분석가이자 데이터 스토리텔링 전문가다.
나는 경영진 보고용 분석 보고서를 작성해야 하는 상품전략팀장이다.

첨부한 CSV 파일을 분석하여, 자동차 모델의 가격, 성능, 연비, 차체 특성 간 관계를 입체적으로 설명하는 인터랙티브 보고서를 작성해 줘.
보고서는 Python 기반으로 작성하되, 시각화는 Plotly를 최대한 활용해 줘.
특히 Plotly Express와 graph_objects를 적절히 병행해서, 단순 차트 나열이 아니라 “의사결정에 도움이 되는 분석 보고서”가 되도록 구성해 줘.

[분석 목표]
1. 가격(price)에 영향을 주는 핵심 요인을 찾는다.
2. 연비(city-mpg, highway-mpg)와 엔진/차체 특성의 관계를 설명한다.
3. 브랜드(make), 차체유형(body-style), 구동방식(drive-wheels), 연료방식(fuel-type)별 차이를 비교한다.
4. 고가 차량군 / 가성비 차량군 / 고성능 차량군을 구분할 수 있는 인사이트를 도출한다.
5. 경영진이 바로 이해할 수 있도록 핵심 요약, 표, 차트를 함께 제시한다.

[데이터 처리 지침]
1. CSV를 불러온 뒤, 실제 데이터가 아닌 설명행/메타행이 있으면 제거해 줘.
2. 컬럼명 공백, 자료형 오류, 숫자형 컬럼의 문자열 문제를 정리해 줘.
3. 결측치는 변수 특성에 따라 처리 기준을 먼저 제안하고, 그 기준대로 정제해 줘.
   - 수치형: 중앙값 또는 그룹별 대표값 기반
   - 범주형: 최빈값 또는 “Unknown” 처리
4. 분석 전에 각 컬럼의 의미를 요약하고, 수치형/범주형으로 구분해 줘.
5. 파생변수를 만들 수 있으면 추가해 줘. 예:
   - 가격 대비 마력(price_per_hp 또는 hp_per_price)
   - 연비 평균(avg_mpg)
   - 차체 크기 지표(길이*너비 등)
   - 엔진 효율 관련 비교지표

[보고서 구성]
아래 순서대로 보고서를 작성해 줘.

1. Executive Summary
- 가장 중요한 인사이트 5가지를 bullet로 먼저 제시
- 어떤 변수들이 가격과 가장 관련이 큰지 요약
- 어떤 세그먼트가 차별화되는지 요약

2. Data Overview
- 데이터 건수, 변수 수, 결측 현황 요약
- Plotly Table로 주요 변수 요약표 제시
- 필요하면 브랜드별/차체유형별 빈도 Bar Chart 제시

3. Univariate Analysis
- 가격, 엔진크기, 마력, 연비, 차체 크기 등의 분포를 보여줘
- 가능한 경우 Plotly 히스토그램 또는 Bar 기반 요약 시각화 활용
- 범주형 변수는 브랜드/차체유형/연료유형/구동방식 비중을 비교해 줘
- Pie Chart는 전체 비중을 빠르게 보여줄 때만 제한적으로 사용해 줘

4. Bivariate / Multivariate Analysis
다양한 Plotly 그래프를 적극 활용해서 아래를 분석해 줘.
- Scatter Plot:
  - x=engine-size, y=price
  - x=horsepower, y=price
  - x=curb-weight, y=price
  - x=avg_mpg, y=price
- Scatter에는 가능한 경우 다음을 적극 반영해 줘.
  - color=make 또는 body-style
  - size=horsepower 또는 engine-size
  - symbol=drive-wheels 또는 fuel-type
  - trendline="ols" 추가
  - facet_col 또는 facet_row를 활용해 브랜드군/연료유형별 비교
- 관계 해석 시 단순 상관이 아니라 “어떤 그룹이 추세선에서 벗어나는지”도 설명해 줘.

5. Comparative Analysis
- 브랜드별 평균 가격, 평균 마력, 평균 연비를 Bar Chart로 비교
- 차체유형(body-style)별 가격/연비 차이를 비교
- 구동방식(drive-wheels)별 성능과 가격 차이를 비교
- 필요하면 grouped bar, stacked bar, horizontal bar를 적절히 선택해 줘
- 상위/하위 브랜드 TOP N 비교 차트도 포함해 줘

6. Relationship Structure
- 수치형 변수 간 상관관계를 요약표로 정리
- 특히 price와 관련 있는 주요 변수 TOP 5를 제시
- 단순 상관계수만 제시하지 말고 해석까지 붙여 줘
- line chart가 유의미한 경우, 정렬된 가격 구간 또는 엔진크기 구간에 따른 평균 지표 변화를 선 그래프로 보여줘

7. Segmentation / Insight
- 가격, 마력, 연비 기준으로 차량군을 3~4개 세그먼트로 나눌 수 있으면 나눠서 설명해 줘
- 예: 고가-고성능형 / 저가-실속형 / 중간가격-균형형
- 세그먼트별 대표 브랜드와 특징을 표와 차트로 보여줘
- 경영적으로 어떤 포지셔닝 전략이 가능한지 제안해 줘

8. Final Recommendation
- 가격 결정에 중요한 요인
- 연비와 성능의 트레이드오프
- 브랜드/차체유형/구동방식별 시사점
- 상품기획/마케팅/영업 관점에서 각각 1~2개 실행 제안

[시각화 지침]
1. Plotly를 최대한 다양하게 활용해 줘.
2. 반드시 아래 유형을 가능한 범위에서 최대한 포함해 줘.
- Scatter Plot
- Line Chart
- Bar Chart
- Pie Chart
- Table
- graph_objects 기반 복수 trace 차트
3. Scatter/Line에서는 아래 기능을 우선 검토해 줘.
- color
- size
- symbol
- line_dash
- trendline
- facet_col / facet_row
- update_traces / update_layout
4. 차트마다 아래를 포함해 줘.
- 제목(title)
- 축 이름(axis label)
- hover 정보
- 해석 2~4문장
5. 불필요하게 같은 메시지의 차트를 반복하지 말고, “서로 다른 질문에 답하는 차트” 위주로 구성해 줘.
6. Pie Chart는 보조적 용도로만 사용하고, 비교가 중요한 경우에는 Bar Chart를 우선 사용해 줘.
7. 경영진 보고용이므로 시각적으로 깔끔하고, 범례와 라벨이 명확해야 한다.
8. 각 차트 아래에 “이 차트로 무엇을 판단할 수 있는지”를 짧게 써 줘.

[출력 형식]
아래 형식으로 결과를 작성해 줘.

- 1부. 데이터 개요
- 2부. 핵심 인사이트 5가지
- 3부. 차트별 분석 결과
- 4부. 종합 결론 및 실행 제안
- 5부. 부록: 주요 표(Table)와 변수 설명

- facet, marker style, multiple trace를 활용해 차트를 최대한 풍부하게 구성해 줘.
- 보고서는 “데이터 분석 결과 + 시각화 이유 + 비즈니스 해석”이 함께 드러나야 한다.
```
