- 주의: 아래 내용에는 `공식 발표`, `방송자가 소개한 사례`, `방송 중 추정/루머`가 함께 포함되어 있다. 특히 GPT-5.5, 내부 엔드포인트 활용, 일부 성능 비교는 방송 시점의 관찰과 추정이므로 별도 검증이 필요하다.

## 한눈에 보는 핵심 요약

이번 방송의 핵심은 `AI 제품 경쟁이 단순 모델 성능 경쟁을 넘어 디자인, 문서, 영상, 브라우저 자동화, 코딩 워크스페이스, 로컬 오픈소스 모델, 로보틱스까지 전면 확장되고 있다`는 점이다. 특히 Anthropic은 `Claude Opus 4.7`, `Claude Design`, `루틴`, `데스크탑 앱 업데이트`로 생산성 전반을 넓히고 있고, OpenAI는 `차기 GPT-5.5 루머`, `이미지 생성 품질 향상`, `Codex 앱 대규모 업데이트`, `에이전트 SDK`로 맞대응하고 있다. Google은 `Gemini 앱`, `AI Studio 디자인 프리뷰`, `Flow Music`, `Chrome 내 Gemini 통합`으로 생태계를 강화하고 있으며, 중국권에서는 `Qwen 3.6`, `GLM`, `Super Gemma 4` 등 오픈소스 진영의 추격 속도가 매우 빠르게 나타난다. 방송 전체 메시지는 `이제 AI는 단일 챗봇이 아니라 실제 업무를 수행하는 도구 집합이며, 코딩-디자인-영상-문서-자동화가 하나의 워크플로우로 수렴하고 있다`는 것이다.

## 1. 보안 및 공급망 공격 이슈

### 1-1. 워드프레스 플러그인 공급망 공격

- 누군가 워드프레스 플러그인 30개 이상을 인수하거나 장악한 뒤 백도어를 심었다는 사례가 소개됐다.
- 워드프레스는 2026년 기준 전 세계 웹사이트의 약 42~43%를 차지하는 것으로 방송에서 언급됐다.
- 플러그인 생태계 기반 CMS 구조에서는 인기 플러그인 하나만 오염돼도 대규모 감염이 발생할 수 있다는 점이 강조됐다.
- 워드프레스 측은 일부 플러그인 강제 업데이트와 저장소 폐쇄 조치를 했다고 소개됐다.

### 1-2. CPU-Z 다운로드 링크 변조

- 하드웨어 정보 확인 도구 `CPU-Z`의 다운로드 링크가 일시적으로 악성 파일로 변조된 사례가 언급됐다.
- 정상 프로그램을 공식 웹사이트에서 받더라도 공급망 공격이 발생하면 사용자가 감염될 수 있다는 점에서, 사용자의 개인 주의만으로 해결하기 어려운 영역이라는 문제의식이 제기됐다.

### 1-3. 해석

- 방송자는 `공급망 공격은 앞으로 더 많아질 것`이라고 봤다.
- 이유는 AI로 취약점 탐지, 해킹 자동화, 코드 조작이 쉬워지면서 공격 효율이 높아지고 있기 때문이라는 관점을 제시했다.
- 실무 시사점은 `플러그인·설치형 프로그램·오픈소스 의존 구조에 대한 신뢰 관리`가 더 중요해졌다는 점이다.

## 2. 바이브 코딩 기반 자동화 사례

### 2-1. KT 인터넷 SLA 자동화 도구

- 방송에서 흥미로운 오픈소스 사례로 `KT 인터넷 속도 측정 및 자동 감면 신청 도구`가 소개됐다.
- KT의 최저 속도 보장 제도를 자동화해, 일정 시간 간격으로 속도를 측정하고 기준 미달 시 감면 신청을 자동화하는 방식이다.
- 핵심 포인트는 `원래는 귀찮아서 안 하던 반복 업무를 AI/자동화로 실행 가능하게 만들었다`는 것이다.

### 2-2. 의미

- 이런 사례는 “돈 버는 AI 제품”보다 더 현실적인 `생활형 자동화 프로덕트`의 예시로 제시됐다.
- 방송자는 “귀찮음이 장벽이던 제도를 프로그램으로 뚫는 시대”가 됐다고 해석했다.

## 3. Anthropic 업데이트

### 3-1. Claude Opus 4.7

- `Claude Opus 4.7`이 공개됐고, 방송에서는 코딩 성능과 실제 업무 활용 성능이 크게 개선됐다고 소개됐다.
- 에이전틱 코딩, 문서 근거 탐색, 장기 맥락 활용, 멀티모달 입력 해석, 사무 업무 벤치마크에서 향상이 있었다고 설명됐다.
- 방송 내 정리 기준으로는 당시 `종합 1위 모델`로 평가됐다.

### 3-2. 특징

- 지시사항 준수 능력이 더 좋아졌다고 하며, 그 결과 기존 프롬프트가 4.7에서는 다르게 작동할 수 있다는 점이 언급됐다.
- 이미지 읽기 성능이 향상돼 더 큰 해상도, 복잡한 화면, 다이어그램 처리에 강해졌다고 소개됐다.
- 실제 업무형 작업, 재무 분석, 지식 노동 벤치마크에서 강점이 크다고 강조됐다.
- 메모리 활용도 향상됐다고 정리됐다.

### 3-3. 비용 이슈

- 공식 가격 자체는 동결됐지만, 토크나이저 변경으로 인해 실제 사용 토큰 수가 늘어날 수 있다는 점이 소개됐다.
- 방송에서는 영어/코드 중심 콘텐츠에서 토큰 증가가 크고, 한국어/중국어/일본어는 상대적으로 변화가 적다는 분석을 전달했다.
- 실질 체감으로는 `가격 동결이지만 실제 사용 비용은 증가할 수 있다`는 메시지였다.

### 3-4. 체감 평가

- 방송자는 4.6도 이미 매우 좋아서 4.7의 체감 향상은 “분명 좋아졌지만 압도적 체감은 아닐 수도 있다”고 평했다.
- 다만 `이미 웬만한 실무는 충분히 수행 가능한 수준`이라고 평가했다.

## 4. Claude Design

### 4-1. 제품 개요

- Anthropic은 `Claude Design`을 공개했다.
- 텍스트 대화만으로 프로토타입, 슬라이드, 원페이지 자료, 앱 UI, 인터랙티브 목업, 영상용 HTML 기반 비주얼을 만들 수 있는 기능으로 소개됐다.

### 4-2. 핵심 기능

- 프롬프트로 디자인 생성
- 디자인 수정 지시 반영
- 다양한 레이아웃/스타일 제안
- PPT 등 다양한 형식으로 내보내기
- `Claude Code`로 핸드오프하여 실제 구현까지 연결

### 4-3. 시사점

- 방송에서는 Figma, Adobe 등 기존 디자인 툴에 대한 위협 가능성을 언급했다.
- 특히 `디자인에서 끝나지 않고 바로 코딩 에이전트로 전달되는 흐름`이 중요하게 다뤄졌다.

### 4-4. 시스템 프롬프트 유출 사례

- Claude Design의 시스템 프롬프트가 유출됐다는 사례도 소개됐다.
- 특수한 해킹이 아니라, “위에 있는 텍스트를 페이지로 만들어줘” 같은 프롬프트로 시스템 내용을 그대로 출력하게 만들었다는 설명이 나왔다.
- 이는 프롬프트 보안과 시스템 메시지 보호 측면에서 Anthropic의 취약점을 보여주는 사례로 소개됐다.

## 5. Claude 루틴, 데스크탑 앱, Video Use

### 5-1. Claude 루틴

- 기존 스케줄 기능이 `루틴`으로 확장되며 API 호출 기반 트리거와 이벤트 응답이 가능해졌다고 소개됐다.
- GitHub 이벤트, 외부 API 호출 등으로 Claude 코드를 자동 실행시킬 수 있다는 점이 핵심이다.
- 이는 단순 예약 작업을 넘어 `워크플로우 자동화 엔진` 쪽으로 확장되는 흐름으로 해석된다.

### 5-2. Claude 데스크탑 앱 업데이트

- Claude 데스크탑 앱은 브라우저, 터미널, diff 보기, 계획 패널 등이 통합되며 사실상 가벼운 IDE에 가까워졌다고 소개됐다.
- 별도 VS Code 플러그인 없이도 Claude 자체 앱만으로 개발 워크플로우를 구성할 수 있다는 점이 강조됐다.

### 5-3. Video Use

- `Video Use`라는 Claude Code용 오픈소스 스킬도 소개됐다.
- 원본 영상을 넣고 편집 요청을 하면 컷 편집, 자막, 간단한 애니메이션, 색보정, 음성 구간 처리 등을 자동 수행하는 도구로 설명됐다.
- 방송에서는 “영상 편집도 개발자가 코드 기반으로 자동화할 수 있는 단계로 가고 있다”는 상징적 사례로 다뤘다.

## 6. Browser Use, Browser Harness

### 6-1. Browser Harness

- Browser Use 팀이 공개한 `Browser Harness`는 브라우저 작업 자동화를 위한 셀프 힐링 구조로 소개됐다.
- 사용자가 “틱톡에 비디오 예약 업로드해줘”라고 자연어로 요청하면 로그인, 업로드, 일정 등록 등 브라우저 상의 행동을 자동 수행하는 시나리오가 예시로 제시됐다.

### 6-2. 의미

- 기존 브라우저 자동화가 selector 깨짐, DOM 변경, 버튼 위치 변화 등에 약했다면, 이 구조는 스스로 수정하며 작업을 지속하는 것이 강점으로 소개됐다.
- 방송에서는 “이제 브라우저에서 하는 거의 모든 일이 자연어 작업으로 바뀔 것”이라는 방향성을 제시했다.

## 7. OpenAI 관련 소식

### 7-1. GPT-5.5 루머

- 방송에서는 차기 OpenAI 모델이 `GPT-5.5` 또는 내부 코드명 `SPUD`로 추정되며, 다음 주 공개 가능성이 높다는 루머를 전했다.
- 근거로는 예측 시장(폴리마켓) 확률과 커뮤니티 반응이 제시됐다.
- 단, 이는 공식 발표가 아닌 방송 시점의 추정이다.

### 7-2. GPT 이미지 생성 품질 향상

- 방송에서는 GPT의 차기 이미지 생성 모델, 또는 차세대 품질로 보이는 결과가 Arena 및 ChatGPT 상에서 관찰되고 있다고 소개했다.
- 특징으로는 다음이 강조됐다.
- 한글 텍스트가 거의 깨지지 않음
- 인물 생성 품질 향상
- PPT, 광고 포스터, 카카오톡 화면, 모바일 앱 UI, 만화풍 이미지 등의 정밀도가 높음
- 기존 Flux 계열보다 텍스트 정확성이 훨씬 우수함

### 7-3. Codex 대규모 업데이트

- `Codex 앱`도 크게 업데이트됐다고 소개됐다.
- 핵심 포인트는 다음과 같다.
- 컴퓨터 유즈: 실제 앱을 클릭·조작하면서 테스트 가능
- 브라우저 완전 제어
- 디자인/이미지 생성과 코딩 흐름 통합
- 90개 이상 플러그인 연동
- 메모리와 장기 작업 지속성
- 데스크탑 기반 개발 워크스페이스화

### 7-4. Xcode 및 모바일 앱 개발 연계

- Exa/X4 계열 툴과 연동해 iOS 시뮬레이터를 띄우고 앱을 만드는 데모도 소개됐다.
- 이는 코덱스가 웹사이트 생성 도구를 넘어 `앱 개발 워크스페이스`로 확장되고 있다는 사례로 설명됐다.

### 7-5. OpenAI Agent SDK

- OpenAI는 샌드박스 환경에서 동작하는 에이전트를 구축할 수 있는 `Agent SDK` 차세대 버전도 공개했다고 소개됐다.
- 파일 검사, 명령 실행, 코드 편집, 장기 작업 수행 등을 안전한 별도 환경에서 다루는 에이전트를 코드 레벨에서 만들 수 있게 해 주는 SDK로 설명됐다.

### 7-6. Excel용 ChatGPT

- 엑셀 안에서 ChatGPT를 불러와 데이터 정리, 조작, 프롬프트 기반 작업을 수행하는 형태의 플러그인/연계 기능도 소개됐다.
- 방송자는 이를 Microsoft Copilot과 직접 경쟁하는 흐름으로 해석했다.

### 7-7. 비공식 OpenAI API 접근 프로젝트

- `OpenAI OSS` 또는 유사한 이름의 비공식 프로젝트가 소개됐다.
- ChatGPT 구독 사용자가 Codex CLI 내부 엔드포인트를 활용해 별도 API 크레딧 없이 API처럼 쓰는 우회 구조로 설명됐다.
- 방송에서도 `공식 지원이 아니며 계정 제재 가능성이 있으므로 개인 실험 수준에서만 조심해서 사용해야 한다`고 언급했다.

## 8. Google 관련 소식

### 8-1. AI Studio 디자인 프리뷰

- Google AI Studio 안에 `디자인 프리뷰` 기능이 들어왔다고 소개됐다.
- 프롬프트를 기반으로 여러 디자인 시안을 제시하고, 그중 하나를 선택해 구현하는 형태다.
- 이는 기존 Stitch 같은 별도 제품의 기능이 AI Studio 안으로 통합되는 흐름으로 해석됐다.

### 8-2. Gemini 맥 앱

- Gemini도 드디어 Mac 앱이 나왔다고 소개됐다.
- 단축키 호출, 현재 화면 맥락 활용, 이미지·비디오·음악 생성 도구와의 연결, NotebookLM/Drive/Photos와의 통합 등이 강점으로 제시됐다.

### 8-3. Flow Music

- Google은 `Flow Music`이라는 음악 생성 제품도 공개했다고 소개됐다.
- 프롬프트 기반 곡 생성, 가사 생성, 뮤직비디오 생성까지 지원하는 것으로 설명됐다.
- 방송자는 아직 Suno 대비 품질은 아쉽지만, “구글이 점점 스노우/수노 영역까지 침범하고 있다”고 해석했다.

### 8-4. Chrome 내 Gemini Skills

- 크롬 브라우저 안의 Gemini 통합 기능도 소개됐다.
- 사용자가 자주 쓰는 프롬프트를 `스킬`로 저장해 다시 호출하는 형태이며, 현재 페이지를 기반으로 바로 질문할 수 있다.
- 브라우저 직접 조작까지는 아니지만, 맥락 기반 보조 도구로는 유용하다고 설명됐다.

## 9. 오픈소스 모델 경쟁

### 9-1. Gemma 4와 로컬 실행

- Gemma 4를 로컬에서 Codex CLI와 연결해 사용하는 흐름이 소개됐다.
- 맥북 M4 Pro 급 장치에서도 로컬 모델 활용이 가능하며, 프라이버시·비용 면에서 장점이 있다는 설명이 있었다.

### 9-2. Super Gemma 4

- 한국 개발자가 만든 `Super Gemma 4`도 소개됐다.
- 특징은 다음과 같다.
- Gemma 4 26B 기반
- 검열 완화
- 한국어 성능 향상
- MLX 최적화
- 4비트 양자화
- Ollama에서도 사용 가능

### 9-3. Qwen 3.6

- Alibaba의 `Qwen 3.6 35B`가 공개됐고, Gemma 4를 여러 벤치에서 앞서는 결과가 소개됐다.
- 방송에서는 `이제 중국 오픈소스 모델이 Sonnet 4.5급까지 따라왔다`는 식의 강한 표현으로 평가했다.
- 핵심 메시지는 `오픈소스가 빠르게 프론티어 모델을 추격 중이며, 로컬 실행 가능한 고성능 모델 시대가 이미 시작됐다`는 것이다.

### 9-4. GLM 5.1

- 중국권 오픈소스 모델인 `GLM 5.1`도 순위표에서 매우 높은 위치를 차지하고 있다고 소개됐다.
- 이 역시 “프론티어 대비 큰 격차가 줄어들고 있다”는 근거로 활용됐다.

## 10. Grok 4.3

### 10-1. 모델 소개

- `Grok 4.3`도 출시됐다고 소개됐다.
- 웹사이트, 문서, 프레젠테이션, 스프레드시트 생성 등 생산성 기능이 강화되었다고 설명됐다.
- 특히 프론트엔드 생성과 문서 생성 능력이 강조됐다.

### 10-2. 평가

- 방송자는 X(구 Twitter) 생태계와의 연결성, 일론 머스크의 적극적인 리트윗 문화 때문에, Grok 기반으로 무엇인가를 만들고 홍보하면 바이럴 효과가 클 수 있다고 해석했다.

## 11. ByteDance와 멀티모달/비디오

### 11-1. Seedance 2.0 API

- `Seedance 2.0`이 API 형태로 풀렸다고 소개됐다.
- 이제 외부 서비스, 제품, 자동화 시스템에 직접 연결해 사용할 수 있는 기반이 마련됐다는 의미다.

### 11-2. OmniShow

- ByteDance의 `OmniShow` 연구는 사람과 제품이 자연스럽게 상호작용하는 UGC 스타일 광고 영상을 생성하는 방향으로 소개됐다.
- 이는 틱톡 스타일의 제품 홍보 영상, AI 인플루언서형 광고, 제품 시연 자동화의 기반 기술로 해석됐다.

## 12. 월드 모델 및 3D

### 12-1. Hunyuan World 2.0

- 이미지나 영상 기반으로 탐험 가능한 3D 월드를 생성하는 오픈소스 모델로 소개됐다.

### 12-2. NVIDIA Lyra 2.0

- 스트리트뷰 한 장만으로 3D 탐험 장면을 만들거나, 공간 시뮬레이션에 활용할 수 있는 가능성이 언급됐다.

## 13. SVG 및 벡터 생성

- 방송에서는 `Lora 1.1`, `1.1 Max` 계열로 보이는 SVG/벡터 생성 모델도 소개됐다.
- 핵심은 단순 래스터 이미지가 아니라 `실제 벡터 기반 결과`를 매우 정교하게 생성할 수 있다는 점이다.
- 로고, 의류 디자인, 아이콘, UI 요소 등을 직접 편집 가능한 벡터 형태로 만들어 줄 수 있다는 점이 강조됐다.

## 14. AI 인덱스 리포트 2026

### 14-1. 방송에서 정리한 핵심 포인트

- AI 성능 향상은 아직 끝나지 않았고 계속 가속 중
- 미국과 중국의 프론티어 모델 격차가 크게 줄어듦
- 생성형 AI의 채택 속도는 인터넷·PC 도입보다 빠른 수준
- 생산성 증가와 초급 일자리 감소가 같은 영역에서 동시에 발생
- 미국 개발자 초급 고용은 감소, 고급 인력 수요는 유지 또는 증가
- 교육 현장은 변화 속도를 따라가지 못하고 있음

### 14-2. 의미

- 방송자는 이 리포트를 근거로 `교육과 채용 기준이 바뀌어야 한다`고 강조했다.
- 단순 알고리즘 문제풀이보다 실제 제품 구현과 가치 창출이 더 중요한 시대라고 해석했다.

## 15. 백준 온라인 저지 서비스 종료와 개발자 평가 방식 변화

### 15-1. 사건

- 백준 온라인 저지 서비스 종료 소식이 소개됐다.
- 공식 종료 이유는 명확히 분석되지 않았지만, 방송에서는 AI 시대와 맞물린 환경 변화가 배경일 수 있다고 해석했다.

### 15-2. 방송자의 관점

- 이제 AI가 ICPC급 알고리즘 문제도 매우 잘 풀기 때문에, 알고리즘 문제풀이 능력을 개발자 실력의 핵심 평가 지표로 보기 어려워졌다는 시각을 제시했다.
- 대신 “실제로 제품을 만들고 사용자 가치를 만드는 능력”이 더 중요해졌다고 강조했다.

## 16. 전체 시사점

### 16-1. 제품 경쟁의 중심이 바뀌고 있다

- 예전에는 “가장 똑똑한 모델”이 핵심이었다면, 지금은 “어떤 워크플로우를 끝까지 자동화하느냐”가 더 중요해지고 있다.
- 코딩, 문서, 디자인, 영상, 브라우저 조작, 이미지 생성, 메모리, 자동화가 하나로 붙고 있다.

### 16-2. 툴 간 수렴 진화가 빨라지고 있다

- Claude, OpenAI, Google, xAI, 중국 오픈소스 모두 서로 비슷한 방향으로 기능을 흡수하고 있다.
- 디자인 툴은 코딩 툴로, 코딩 툴은 문서·브라우저·컴퓨터 유즈 툴로 확장 중이다.

### 16-3. 오픈소스와 로컬 모델의 전략 가치가 커진다

- 이제 고성능 모델을 로컬에서 돌리고, 검열 완화 버전·한국어 최적화 버전·양자화 버전으로 변형하는 흐름이 활발하다.
- 이는 비용 절감뿐 아니라 데이터 통제권, 커스터마이징, 민감 업무 처리 측면에서 장점이 크다.

### 16-4. 교육과 채용 기준이 바뀌어야 한다

- 방송 전체의 암묵적 메시지는 “AI 시대 개발자 역량은 문제풀이 정답률보다 실제 프로덕트 구현력, 자동화 설계력, 사용자 가치 창출력으로 옮겨가고 있다”는 것이다.

## 결론

이번 방송은 단순한 주간 뉴스 모음이 아니라, `AI 산업이 어디로 수렴하고 있는지`를 보여주는 강한 신호를 담고 있었다. Anthropic은 디자인과 코딩 자동화를 확장하고, OpenAI는 워크스페이스형 Codex와 차기 모델 루머로 기대를 높이고, Google은 앱·브라우저·음악·디자인까지 생태계를 넓히고 있다. 동시에 오픈소스 진영은 Qwen, GLM, Gemma 파생 모델을 통해 프론티어와의 격차를 빠르게 줄이고 있다. 정리하면 현재 AI 업계의 핵심 화두는 `더 똑똑한 단일 모델`이 아니라 `실제 업무를 끝까지 연결하는 통합형 실행 도구`이며, 앞으로의 경쟁은 모델 성능 자체보다 `워크플로우 장악력`에서 더 치열해질 가능성이 높다.
