# 2026-04-23 DXAX 수업 한페이지 요약

## 수업 전체 요약
이번 수업은 오전과 오후의 주제가 비교적 뚜렷하게 나뉘었다. 오전에는 전날 이어진 협업형 AX 흐름 위에서 Notion, Slack, MS Teams 같은 협업 소프트웨어를 어떻게 볼 것인지, 그리고 Codex 앱의 메모리 구조, AGENTS/행동지침, 스킬, MCP, 로컬 파일 관리가 왜 중요한지를 설명했다. 또한 카카오톡 API 연동 실습과 JSON 바디/헤더 디버깅을 통해 협업 메시지와 자동화 파이프라인이 실제로 어떻게 연결되는지를 보여 줬다. 이 과정에서 진짜 AX는 단순히 사람이 더 빨라지는 것이 아니라, 반복 프로세스를 알고리즘 워크플로우로 재구성하고 `human in the loop`를 남겨 안정적으로 운영하는 것이라고 정리했다.

오후에는 무게중심이 명확하게 이미지 생성으로 넘어갔다. 이미지·영상 생성 AI가 어떤 원리로 동작하는지, LLM과 디퓨전 모델이 어떻게 다른지, 왜 CLIP 같은 구조가 필요한지를 설명했고, Stable Diffusion, Flux, Nanobana, Ductape 같은 이미지 생성 툴과 모델들을 비교했다. 이어서 네거티브 프롬프트, 비율, 해상도, 업스케일링 같은 실무 팁을 다루고, GPT/Codex를 통해 PPT용 인포그래픽, 마스터 이미지, 동일한 마스코트 시리즈처럼 실제 발표자료에 바로 쓸 수 있는 시각 자료를 생성하는 실습까지 진행했다. 즉 이날 수업은 오전의 협업·자동화 운영과 오후의 이미지 생성 실습이 반반이 아니라, 오후에 이미지 생성 파트가 별도 주제로 크게 전개된 날이라고 보는 것이 맞다.

## 무엇을 배웠나
- 오전에는 Notion, Slack, MS Teams 같은 협업 도구를 AX 관점에서 어떻게 선택하고 활용할지 살펴봤다.
- 협업 환경에서는 문서, 채팅, 데이터베이스가 AI가 읽기 좋은 형식으로 정리되어야 한다는 점을 다시 확인했다.
- Codex 앱은 단순 채팅 도구가 아니라 컨텍스트, 메모리, 행동지침, 스킬을 묶어 운영하는 에이전트 허브라는 점을 배웠다.
- 메모리 구조와 AGENTS/행동지침 파일을 활용하면 반복 프롬프트를 줄이고 더 일관된 작업 수행이 가능하다는 점을 이해했다.
- 카카오톡 API 연동 실습을 통해 JSON 바디 구조, 헤더, 파라미터 오류를 디버깅하는 과정을 배웠다.
- `human in the loop` 구조와 업무 정량화를 통해 AX 효과를 운영과 수치 관점에서 점검하는 방식을 배웠다.
- 오후에는 이미지 생성 AI의 기본 원리와 LLM 기반 생성과 디퓨전 기반 생성의 차이를 배웠다.
- CLIP 개념을 통해 이미지 생성 모델이 텍스트를 어떻게 이해하고 원하는 그림 방향으로 맞추는지 큰 흐름을 이해했다.
- Stable Diffusion, Flux, Nanobana, Ductape 등 이미지 생성 툴과 모델의 차이를 비교했다.
- 네거티브 프롬프트, 화면 비율, 해상도, 업스케일링 같은 이미지 생성 실무 팁을 익혔다.
- 발표자료용 인포그래픽, 마스터 이미지, 마스코트 시리즈처럼 실제 업무 결과물에 바로 쓰는 이미지 제작 실습을 진행했다.

## 핵심 메시지
- 오전의 핵심은 협업형 AX를 위해 도구보다 구조를 먼저 설계해야 한다는 점이었다.
- Codex를 잘 쓰려면 프롬프트 기술보다 컨텍스트, 메모리, 행동지침, 스킬 구조를 먼저 갖춰야 한다.
- 진짜 효율화는 사람이 더 빨리 일하는 것이 아니라, 반복 프로세스 자체를 자동화 가능한 워크플로우로 바꾸는 것이다.
- 오후의 핵심은 이미지 생성도 단순 재미가 아니라, 발표자료와 커뮤니케이션 자산을 만드는 실무 도구라는 점이었다.

## 기억해야 할 디테일
- Notion은 협업 문서와 데이터베이스 구조를 함께 운영하기 좋고, Slack은 AI 친화적인 채널형 협업에 강점이 있다.
- 카카오톡은 보안과 접근 제약이 강한 편이라 업무 자동화에 붙일 때 인증과 구조를 더 신중히 다뤄야 한다.
- Codex의 메모리 구조와 AGENTS/행동지침 파일은 반복 프롬프트 작성을 줄이고 일관된 결과를 만드는 데 유용하다.
- 스킬은 프롬프트 템플릿이 아니라, 특정 작업을 안정적으로 수행하게 만드는 재사용 가능한 작업 규칙 묶음이다.
- 카카오 API 실습에서는 JSON 바디 형식, 헤더 누락, 파라미터 타입 문제 같은 작은 구조 오류가 전체 흐름을 깨뜨릴 수 있음을 확인했다.
- `human in the loop`는 자동화를 포기하는 개념이 아니라, 사람 검토 지점을 전략적으로 남겨 운영 안정성을 높이는 방식이다.
- 이미지 생성에서는 LLM과 같은 방식이 아니라 디퓨전과 CLIP 같은 별도 구조가 작동한다는 점을 구분해야 한다.
- 이미지 생성에서는 네거티브 프롬프트 같은 전통 기법보다, 최근에는 자연어 요청과 모델 선택이 더 중요해졌다는 점을 설명했다.
- 해상도는 처음부터 무작정 높이기보다 낮은 해상도로 생성 후 업스케일링하는 방식이 효율적일 수 있다.

## 결론
2026년 4월 23일 수업은 오전과 오후가 분명히 다른 축을 가진 날이었다. 오전에는 협업형 AX를 실제 운영 구조로 옮기는 방법을, 오후에는 이미지 생성 AI를 이해하고 발표자료용 시각 자산을 만드는 방법을 집중적으로 다뤘다. 따라서 이 수업의 핵심은 협업 자동화와 이미지 생성 실습을 각각 별도의 실무 주제로 이해하는 것이며, 특히 오후 파트는 단순 부가 내용이 아니라 수업의 큰 절반을 차지한 메인 주제였다.
