# 2026-04-16 DXAX 수업 한페이지 요약

## 수업 전체 요약
이날 수업의 핵심은 `요약`에서 한 단계 더 나아가, **데이터를 읽고 분석하고 시각화한 뒤 비즈니스 인사이트와 보고서로 전환하는 실무 흐름**을 익히는 것이었다. 오전에는 데이터 기반 사고, 정형·반정형·비정형 데이터의 차이, 데이터 리터러시와 실무자 역할 변화를 설명했고, 오후에는 CSV·Excel 데이터를 Codex와 GPT로 분석하며 그래프 생성, 해석, 검증, 원페이지 보고서 작성까지 이어지는 과정을 실습했다. 마지막에는 NotebookLM을 이용해 결과를 PDF나 슬라이드 초안 형태로 다듬는 흐름까지 연결했다.

## 무엇을 배웠나
- 전날 다룬 AI 검색, wiki LLM, 로컬 Whisper, 원페이지 요약 흐름을 다시 복습하면서 결과물은 반드시 사람이 검증해야 한다는 점을 먼저 짚었다.
- `DX·AX 시대의 핵심 자산은 데이터`라는 관점에서 데이터 기반 사고와 DDD(Data Driven Decision)를 설명했다.
- AI의 성능을 좌우하는 것은 모델 이름보다 `양질의 데이터`이며, 결국 실무자는 데이터를 이해하고 관리할 수 있어야 한다는 점을 강조했다.
- 정형 데이터(Excel, CSV, DB), 반정형 데이터(JSON, 태그 구조 문서), 비정형 데이터(PDF, 이미지, 댓글, 음성)를 구분하고, 이제는 비정형 데이터도 LLM으로 처리 가능한 실무 자산이 되었다고 설명했다.
- 데이터 위에 정보, 지식, 인사이트가 쌓인다는 흐름을 통해 실무자의 역할이 단순 집계자가 아니라 `인사이트 발굴자`로 이동하고 있음을 설명했다.
- 데이터 정제 단계에서 결측치, 이상치, 타입 오류를 AI로 빠르게 처리할 수 있지만, 적절한 보정인지 여부는 사람 판단이 필요하다고 강조했다.
- 시각화에서는 선 그래프, 막대 그래프, 원 그래프, 히스토그램, 스캐터 플롯 등 그래프 유형을 목적에 맞게 선택해야 한다는 점을 다뤘다.
- Codex와 GPT를 통해 Python 코드를 직접 짜지 않고도 분석과 시각화를 수행할 수 있음을 실습으로 확인했다.
- Orange3에서 했던 흐름을 더 확장해, Codex가 직접 패키지 설치와 스크립트 실행까지 이어가며 분석 환경을 구성하는 방식도 보여줬다.
- 그래프는 자동 생성할 수 있어도, 축 왜곡, 특정 기간만 발췌하는 편향, 상관관계와 인과관계 혼동 같은 해석 오류는 사람이 비판적으로 검토해야 한다고 설명했다.
- 판매 데이터셋을 예시로 기술통계, 추세 분석, 평균 주문 금액, 매장별/결제수단별 패턴 등을 읽고, 거기서 실제 비즈니스 액션을 도출하는 과정을 실습했다.
- 분석 결과를 다시 `비즈니스 언어`로 번역해 이메일형 보고, 경영진용 원페이지, 실행 제안 문장으로 바꾸는 방법을 다뤘다.
- NotebookLM은 분석 결과를 더 읽기 쉬운 PDF나 슬라이드형 정리 자료로 다듬는 보조 도구로 활용됐다.

## 핵심 메시지
1. **AI가 그래프를 그려주는 것이 목표가 아니다.**
핵심은 실무자가 데이터의 의미를 읽고, 왜 그런 숫자가 나왔는지 설명하고, 다음 행동까지 제안하는 것이다.

2. **데이터 리터러시는 AX 시대의 기본 역량이다.**
무엇을 물어볼지, 무엇이 이상한지, 어떤 해석이 위험한지 판단하려면 데이터 구조와 맥락을 이해해야 한다.

3. **분석 결과는 검증과 번역이 필요하다.**
자동 분석 결과를 그대로 쓰는 것이 아니라, 왜곡 여부를 확인하고 조직이 이해할 언어로 바꾸어야 한다.

4. **원페이지 보고서는 숫자 나열이 아니라 스토리 구조다.**
배경, 목적, 핵심 발견, 시사점, 액션 제안이 함께 있어야 설득력 있는 보고서가 된다.

## 기억해야 할 디테일
- 정형 데이터는 빠르고 분석이 쉽지만, 비정형 데이터도 이제는 충분히 활용 가능한 자산이다.
- AI는 결측치 보정, 그래프 생성, 기초 통계 계산을 잘하지만 해석 책임은 사람에게 남는다.
- 그래프를 볼 때는 축 범위, 기간 선택, 비교 기준 누락 여부를 꼭 확인해야 한다.
- 상관관계와 인과관계를 혼동하면 잘못된 비즈니스 결론으로 이어질 수 있다.
- Codex/GPT는 Python 분석 환경 구성과 시각화까지 자동화할 수 있어, 실무자의 진입장벽을 크게 낮춘다.
- NotebookLM은 최종 보고서 다듬기와 메시지 정리에 유용하지만, 원천 데이터 해석을 대체하지는 않는다.

## 결론
4월 16일 수업은 데이터 분석 도구를 소개하는 시간이 아니라, **실무 데이터에서 인사이트를 뽑아 의사결정 문서로 바꾸는 전 과정을 AI와 함께 수행하는 방법**을 익히는 시간이었다. 결국 AX에서 중요한 것은 AI 사용 자체가 아니라, 데이터를 이해하고 검증하고 비즈니스 언어로 바꾸는 사람의 역할이라는 점이 반복해서 강조되었다.
