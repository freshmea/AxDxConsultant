# 2026-04-13 DXAX 수업 1페이지 요약

## 수업 한줄 요약
이날 수업의 핵심은 **프롬프트를 정교하게 쓰는 것을 넘어, NotebookLM과 회의록/자료 요약 같은 실무형 reasoning 도구를 함께 활용하는 방법**이었다.

## 무엇을 했나
- 금요일 수업에서 다룬 데이터 기반 사고를 다시 복습하며, 모든 업무를 가능하면 데이터 형태로 바꾸고 특히 Markdown/텍스트 형태로 정리해 두는 습관의 중요성을 강조했다.
- 오늘의 본론은 프롬프트 작성법 심화였다. 단순 지시가 아니라 제약 조건, 맥락, 구조, 원하는 결과 형태를 포함한 **실전형 프롬프트 설계**를 실습했다.
- 복잡한 업무 지시는 구조화가 필요하며, 이것이 나중에 AI 에이전트나 Codex 같은 도구 사용으로 이어진다고 설명했다.
- 오전에는 **NotebookLM**을 시연했다. 배경조사 자료를 넣고 블로그, 인포그래픽, PPT, 음성/영상형 결과물까지 빠르게 만드는 흐름을 보여줬다.
- NotebookLM이 단순 요약기가 아니라, 자료를 넣어두고 맥락 기반으로 결과물을 생산하는 강력한 도구라는 점을 반복 강조했다.
- 오후에는 “회의록 정리해줘” 같은 모호한 요청이 왜 좋지 않은지 설명하면서, 회의 성격, 목적, 대상, 필요한 산출물 형태를 포함한 프롬프트 설계가 중요하다고 다시 짚었다.
- 녹취록과 문서 자료를 활용해 회의록 초안, 요약, 인포그래픽, PPT 형태로 바꾸는 실습을 이어갔다.
- 후반에는 **클로바노트**를 소개하며 화자 분리, 회의 녹취, 회의록 초안 작성, 액션 아이템 추출 같은 실제 업무 적용 사례를 설명했다.
- 다만 원본 데이터 품질, 보안 정책, 녹음 동의, 결과물 검토가 반드시 필요하다고 함께 언급했다.

## 핵심 메시지
1. **AI는 검색 엔진이 아니라 reasoning engine처럼 써야 한다.**
자료를 모아 생각을 정리하고, 구조화된 결과물로 바꾸는 데 강점이 있다.

2. **NotebookLM은 실무형 2차 산출물 제작 도구다.**
보고서 초안, 발표자료, 인포그래픽, 음성 기반 설명 자료로 빠르게 확장할 수 있다.

3. **모호한 프롬프트는 모호한 결과를 만든다.**
특히 회의록, 요약, 보고서류는 목적과 형식을 분명히 지정해야 한다.

4. **회의 자동화는 실무에서 바로 체감되는 quick win이다.**
녹취 → 요약 → 액션 아이템 추출은 가장 현실적인 AX 적용 사례 중 하나다.

## 기억해야 할 디테일
- 업무 자료는 가능하면 MD/text로 정리해 두는 것이 좋다
- NotebookLM은 소스를 먼저 넣고 그 위에 질문·산출을 얹는 구조다
- 회의록 프롬프트는 회의 목적, 참석자 관점, 결과 형식까지 지정해야 한다
- 클로바노트는 화자 분리에 강점이 있지만, 최종 요약 품질은 별도 LLM 검토가 더 좋을 수 있다
- 녹음 데이터는 보안 정책과 동의 절차를 확인해야 한다

## 결론
이날 수업은 “프롬프트를 잘 쓴다”를 넘어서, **배경조사 자료와 녹취 데이터를 AI가 바로 활용할 수 있는 작업물로 바꾸는 실무형 흐름**을 체득하게 하는 시간이었다.
