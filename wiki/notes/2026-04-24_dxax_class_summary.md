# 2026-04-24 DXAX 수업 한페이지 요약

## 수업 전체 요약
이번 수업은 3주차 워크플로우 학습을 마무리하면서, 오전에는 `n8n` 중심의 자동화 흐름과 협업 메시지 연동을 다시 정리하고, 오후에는 이미지 생성 AI의 구조와 실습을 본격적으로 다룬 날이었다. 오전에는 Trigger-Action-실행 구조를 다시 짚고, `n8n`이 오픈소스 워크플로우 엔진으로서 어떤 장점이 있는지 설명했다. 이어서 정부 페이지 조회, Slack/카카오톡 알림 연결, 웹훅과 MCP를 통한 Codex-워크플로우 연동, 이미지 생성 API를 작업 흐름 안에 넣는 방식까지 확장했다. 메시지 전송 자동화는 단순 구현보다 권한, 문서, 인증, 디버깅이 더 중요하다는 점도 같이 강조했다.

오후의 중심은 이미지 생성이었다. 이미지 생성이 단순한 부가 기능이 아니라 설득, 홍보, 발표자료, 인포그래픽, 썸네일, 영상 제작의 출발점이 될 수 있다는 관점에서 출발해, LLM과 디퓨전 모델의 차이, CLIP의 역할, 프롬프트 구조, 스타일·재질·비율·해상도·업스케일링 개념을 설명했다. 또한 Stable Diffusion, Flux, Nanobana, Ductape와 같은 모델 흐름을 비교하고, 로컬 모델을 API처럼 연결해 Codex/워크플로우 안에서 호출하는 방식도 소개했다. 실습에서는 콘셉트 이미지, SNS용 라이프스타일 컷, 제품 홍보 이미지처럼 실제 업무 결과물에 가까운 산출물을 만드는 연습을 진행했다.

## 무엇을 배웠나
- 3주차 전체 흐름을 복기하며, 개인 프롬프트 활용에서 워크플로우 기반 AX로 넘어온 이유를 정리했다.
- `n8n`의 Trigger, Action, 실행 구조를 다시 확인하고, 실제 업무 흐름을 알고리즘처럼 설계하는 관점을 배웠다.
- 정부 페이지 조회와 알림 전송 예시를 통해 실무형 워크플로우가 어떻게 구성되는지 살펴봤다.
- Slack과 카카오톡 메시지 연동에서 API 설정, 권한, 문서 확인, 인증 절차가 왜 중요한지 이해했다.
- 웹훅과 MCP를 통해 Codex와 워크플로우 엔진을 연결하는 방식을 배웠다.
- 이미지 생성도 API나 로컬 모델 연결을 통해 자동화 흐름 안에 포함시킬 수 있다는 점을 확인했다.
- 오후에는 이미지 생성 AI와 LLM의 차이, 디퓨전 모델의 특징, CLIP의 역할을 배웠다.
- 프롬프트를 단순 키워드 나열이 아니라 콘셉트, 기능, 재질, 배경, 사용 장면까지 포함한 기획 문장으로 설계하는 방식을 익혔다.
- Stable Diffusion, Flux, Nanobana, Ductape 등 모델과 툴의 차이를 비교했다.
- 스타일라이즈, 인물 생성 오류 점검, 손가락·관절·텍스트 왜곡 같은 QA 포인트를 확인했다.
- 해상도, 비율, CFG, 스텝 수치, 업스케일링 등 로컬 이미지 생성 파라미터가 결과에 어떤 영향을 주는지 살펴봤다.
- 제품 콘셉트 이미지, 인스타그램 홍보 컷, 발표자료용 인포그래픽처럼 실제 업무에 쓰는 시각 자산을 만드는 실습을 진행했다.

## 핵심 메시지
- 워크플로우 AX의 핵심은 기능을 많이 아는 것이 아니라, 업무 흐름을 실행 가능한 구조로 바꾸는 것이다.
- 메시지 자동화와 외부 서비스 연동은 코드보다 권한, 인증, 문서 해석, 디버깅 역량이 더 중요할 때가 많다.
- 이미지 생성은 단순한 그림 놀이가 아니라 기획서, 마케팅, 발표자료를 위한 실무형 산출물 제작 도구다.
- 앞으로의 AX는 문서 자동화와 이미지 생성을 따로 보지 않고, 하나의 워크플로우 안에서 연결하는 방향으로 간다.

## 기억해야 할 디테일
- 카카오톡 자동화는 Slack보다 제약이 많아 무료 범위, 채널 메시지, 특정 대상 발송 권한을 구분해서 봐야 한다.
- 웹훅과 MCP를 잘 연결하면 Codex가 직접 워크플로우를 호출하고 결과를 다시 이어받는 구조를 만들 수 있다.
- 이미지 생성 API를 워크플로우 안에 넣으면 “텍스트 요청 → 이미지 생성 → 후속 업무 사용” 흐름이 가능해진다.
- CLIP은 텍스트와 이미지를 연결해 모델이 원하는 그림 방향을 맞추도록 돕는 핵심 개념이다.
- 로컬 이미지 생성에서는 모델 선택뿐 아니라 CFG, steps, 해상도, 업스케일링, 하드웨어 자원까지 함께 고려해야 한다.
- 인물 생성 결과는 손가락, 관절, 시선, 텍스트 표현, 배경 왜곡을 중심으로 QA하는 습관이 필요하다.
- 최종 이미지는 한 번에 완성하기보다 콘셉트 설계, 디테일 튜닝, 비율/매체 맞춤, QA 순서로 접근하는 편이 안정적이다.

## 결론
2026년 4월 24일 수업은 3주차 AX 학습을 `워크플로우 자동화`와 `이미지 생성 실무`라는 두 축으로 묶어 마무리한 시간이었다. 오전에는 협업 메시지와 외부 도구를 연결하는 자동화 구조를, 오후에는 발표자료와 마케팅에 직접 쓰이는 이미지 생성 구조를 익혔다. 핵심은 이제 AI를 단일 도구로 쓰는 것이 아니라, 텍스트·메시지·이미지 생성까지 하나의 작업 흐름 안에서 연결해 실제 업무 산출물로 이어지게 만드는 것이다.
