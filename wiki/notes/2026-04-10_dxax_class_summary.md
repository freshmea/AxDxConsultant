# 2026-04-10 DXAX 수업 1페이지 요약

## 수업 한줄 요약
이날 수업의 핵심은 **좋은 문제 정의와 좋은 프롬프트가 결국 같은 일이며, 이후에는 데이터를 기반으로 사고하는 방향으로 넘어가야 한다**는 점이었다.

## 무엇을 했나
- 전날 실습을 복습하면서, 문제 정의를 쉽게 말하면 “프롬프트를 제대로 쓰는 방법”이라고 정리했다.
- `쓰레기를 넣으면 쓰레기가 나온다`는 원칙을 설명하며, AI에 넣는 데이터와 배경지식의 품질, 의도 명시의 정확성이 결과 품질을 좌우한다고 강조했다.
- 프롬프트에는 배경지식, 목적, 절차, 도구, 제약 조건, 결과 형식을 명확하게 넣어야 하며, 그렇지 않으면 지나치게 장황하거나 엉뚱한 결과가 나온다고 설명했다.
- 모델 내부 학습 데이터의 영향, 편향 가능성, 모델별 특성 차이도 짚으면서, AI 답변을 비판적으로 검토해야 한다고 강조했다.
- 오전 후반부터는 **데이터 기반 사고**로 전환했다. 업무를 판단의 연속으로 보지 말고, 데이터로 표현하고 질문하는 방식으로 바꿔야 한다고 설명했다.
- “무슨 일이 일어나는가 / 왜 일어나는가 / 앞으로 어떻게 할 것인가”라는 질문 틀로 데이터를 바라보는 방식을 제시했다.
- 오후에는 **Orange3**를 설치하고, 위젯 기반 데이터 흐름, 샘플링, 모델 학습, 타이타닉 데이터셋 예시, 시각화, 모델 저장 등 데이터 분석 입문 실습을 진행했다.
- 마지막에는 Orange를 완벽히 다루는 것이 목적이 아니라, 데이터·샘플링·피처·타겟·모델·검증 같은 용어와 흐름에 익숙해지는 것이 우선이라고 정리했다.

## 핵심 메시지
1. **문제 정의는 프롬프트 설계다.**
AI가 제대로 일하려면 문제 자체를 명확히 번역해서 전달해야 한다.

2. **모델을 맹신하면 안 된다.**
모델 내부 데이터와 편향이 항상 영향을 줄 수 있으므로, 결과를 비판적으로 읽어야 한다.

3. **업무를 데이터 질문으로 바꿔야 한다.**
현상 설명, 원인 추론, 다음 행동 제안은 모두 데이터 중심 사고 위에서 가능해진다.

4. **분석 도구는 직접 다루기보다 흐름을 이해하는 데 먼저 쓴다.**
Orange 실습은 데이터 분석을 완벽히 하기보다, AI와 데이터 업무를 연결하는 직관을 만드는 데 목적이 있었다.

## 기억해야 할 디테일
- 프롬프트 핵심 요소: 배경지식, 목적, 절차, 제약, 출력 형식
- 문제 정의가 흐리면 결과도 흐려진다
- 데이터 기반 질문: `What`, `Why`, `How`
- Orange는 위젯 기반 데이터 흐름 도구이며, 머신러닝 개념을 시각적으로 이해하기 좋다
- 실무 목표는 Orange를 붙잡는 것이 아니라, 나중에 AI에게 더 정확한 분석 지시를 내릴 수 있는 감각을 만드는 것

## 결론
이날 수업은 프롬프트 작성 수업에서 끝나지 않고, **문제 정의 → 데이터 기반 사고 → 분석 도구 이해**로 이어지며 AX 실무의 다음 단계로 넘어가는 연결점 역할을 했다.
