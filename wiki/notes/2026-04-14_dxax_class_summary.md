# 2026-04-14 DXAX 수업 1페이지 요약

## 수업 한줄 요약
이날 수업의 핵심은 **Codex를 중심으로 MCP, 스킬, 딥리서치, NotebookLM을 연결해 실제 문서 작성·이메일 발송·자동화까지 이어지는 AX 실무 흐름**을 익히는 것이었다.

## 무엇을 했나
- 전날 실습을 복습하며, 단순 프롬프트만으로는 부족하고 **딥리서치로 배경조사 → Codex로 도구 사용 → NotebookLM으로 2차 산출물 정리**라는 흐름을 강조했다.
- LLM은 맥락을 오래 기억하지 못하므로, 복잡한 업무일수록 배경조사를 먼저 하고 그 맥락 위에 지시를 얹어야 한다고 설명했다.
- Codex를 사용하는 이유는 **툴 콜링과 MCP**를 통해 파일 읽기/쓰기, 문서 처리, 외부 서비스 연동이 가능하기 때문이라고 정리했다.
- MCP는 웹 서비스, 파일 시스템, 이메일, 문서 도구 등을 AI와 연결하는 통로이며, 이것이 들어가야 비로소 에이전트형 업무 자동화가 가능해진다고 설명했다.
- 수업 중에는 Codex에서 스킬과 앱을 연결해 **이메일 작성 및 실제 발송**, 자동화 탭을 이용한 주기 작업, 문서 작성 지원 등을 시연했다.
- MCP/스킬 구조를 통해 Gmail, Notion, 문서 도구 등과 연결하면 반복 업무를 크게 줄일 수 있다는 점을 보여줬다.
- NotebookLM과의 연동 가능성, 보고서용 자료 정리, 인포그래픽 생성도 다시 언급하며, 하나의 자료를 여러 형식으로 재가공하는 흐름을 이어갔다.
- 후반에는 소설 작성·기획안 작성 같은 긴 문서 생성 실습, 그리고 **레드팀 방식 최종 검수**를 설명했다. AI를 여러 창으로 분리해 비판적으로 검토하게 하면 품질을 높일 수 있다고 했다.
- 마지막에는 회의 준비, 회의록 초안 작성, 액션 아이템 추출, 회의록 프롬프트 템플릿화까지 연결하며, 문서업무 자동화의 실전 사례를 다시 정리했다.

## 핵심 메시지
1. **Codex는 AX 실무의 허브 역할을 한다.**
단순 답변형 챗봇이 아니라, 파일과 서비스에 연결된 실행형 도구로 봐야 한다.

2. **MCP가 들어가야 진짜 자동화가 된다.**
AI가 메일을 보내고, 문서를 읽고, 외부 도구를 호출하려면 연결 구조가 먼저 필요하다.

3. **딥리서치와 도구 사용은 같이 가야 한다.**
좋은 배경조사 없이 실행하면 품질이 떨어지고, 실행 도구 없이 조사만 해도 실무성이 떨어진다.

4. **최종 품질은 검수에서 결정된다.**
레드팀 검수, 예상 질문 생성, 템플릿화가 결과물 완성도를 크게 높여 준다.

## 기억해야 할 디테일
- Codex 자동화는 로컬 실행 기반이라 앱/환경을 켜 두어야 한다
- Gmail 발송, 주기 작업 같은 자동화는 quick win 후보가 될 수 있다
- MCP/스킬은 강력하지만 설정과 권한 이슈가 있으므로 단계적으로 적용해야 한다
- 긴 문서는 `조사 → 초안 → 검수 → 재작성` 흐름으로 가는 것이 안정적이다
- 회의록도 “그냥 요약”보다 목적/형식/추출 항목을 지정한 프롬프트 템플릿이 중요하다

## 결론
이날 수업은 AI를 단순 생성 도구로 쓰는 단계를 넘어, **도구 연결과 자동화를 통해 실제 업무를 수행하게 만드는 AX 운영 방식**을 익히는 시간이었다.
